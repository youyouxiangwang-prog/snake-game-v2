# Snake 3D — 产品定位文档 (PRD)

> 基于 IO 游戏成功案例研究（Slither.io, Agar.io, Diep.io）  
> 版本：1.0 | 日期：2026-04-02

---

## 一、市场现状与机会

### 1.1 IO 游戏的市场验证

| 游戏 | 发布年份 | 存活时长 | 核心机制 | 变现方式 |
|------|----------|----------|----------|----------|
| Slither.io | 2016 | 9年+ 仍在运营 | 蛇吃球长大，撞头即死 | 皮肤购买 |
| Agar.io | 2015 | 10年+ | 吃小球变大，吃其他玩家 | 皮肤 + 会员 |
| Diep.io | 2016 | 8年+ | 坦克升级，射击 | 皮肤 + 颜色 |

**关键洞察：** 所有长寿命 IO 游戏都有一个共同点 — **死亡重开无惩罚**。这是它们成功的底层逻辑。

### 1.2 现有蛇游戏的问题

| 问题 | 描述 |
|------|------|
| 视觉过时 | 2D 画面，没有沉浸感 |
| 操作别扭 | 方向键/WASD 反直觉 |
| 目标模糊 | 没有清晰的目标感 |
| 成长无惊喜 | 分数没有实质奖励 |
| 分享动机弱 | 死了就死了，没有"炫耀"出口 |

### 1.3 我们的机会

**核心差异点：极致视觉 + 收集驱动的成就感**

3D + 现代渲染技术（UnrealBloom, PBR, 粒子系统）是 2D IO 游戏没有的维度。
视觉足够惊艳 → 用户愿意分享截图/视频 → 病毒传播。

## 二、核心体验设计

### 2.1 游戏循环（Core Loop）

```
进入游戏 → 自动引导(3秒) → 核心循环(∞) → 死亡 → 看分数 → 分享/重开
                                    ↑
                  ┌─────────────────┼─────────────────┐
                  ↓                 ↓                 ↓
              收集皮肤          完成挑战          提升段位
```

**核心循环 15 字：** 操作简单、成长可见、死亡无感、分享有面

### 2.2 操作设计

| 要素 | 设计 |
|------|------|
| 操作方式 | **鼠标移动跟随**（鼠标指哪蛇往哪，不需按键） |
| 加速 | 按住鼠标左键加速（消耗能量条） |
| 暂停 | ESC |
| 相当于 | Slither.io 的操作手感，但 3D 沉浸版 |

**为什么鼠标跟随：** 触屏友好（未来移动端），操作直觉，0 学习成本。

### 2.3 视觉体验目标

> "用户第一眼看到游戏，5秒内发出'哇'的声音"

| 效果 | 描述 | 参考 |
|------|------|------|
| 蛇身 | 全息虹彩 + 随角度变色 + 发光粒子尾迹 | 科幻风格 |
| 食物 | 多层发光 + 被吃时爆炸粒子 | 霓虹风格 |
| 场景 | 暗色调 + 雾效 + 边界光环 | 赛博朋克 |
| 吃食物 | FOV 拉伸 + 镜头震动 + 屏幕闪光 | 主机游戏手感 |
| 死亡 | 蛇身碎片爆炸四散 + 慢镜头 1 秒 | 电影级反馈 |

### 2.4 收集系统（核心差异点）

**Skin Collection — 让每次游戏都有成就感**

| 收集内容 | 解锁方式 | 视觉表现 |
|----------|----------|----------|
| 蛇皮肤 | 游戏内货币购买 | 10+ 种不同风格 |
| 背景主题 | 成就解锁 | 5 种不同氛围 |
| 粒子特效 | 段位解锁 | 尾迹颜色/形态变化 |
| 称号 | 累计分数解锁 | 显示在蛇头上方 |

**货币系统：**
- 每次吃食物：+1 金币
- 每次存活 30 秒：+10 金币
- 观看广告（未来）：+50 金币
- **Skin 价格：100-2000 金币不等**

### 2.5 挑战系统（让用户有目标）

| 挑战类型 | 描述 | 奖励 |
|----------|------|------|
| 每日挑战 | 今日目标（如：吃 50 个食物） | 稀有皮肤碎片 |
| 段位赛 | 从青铜到王者的分数要求 | 专属特效 |
| 收集成就 | "第一次吃金色食物"等 | 永久解锁 |

**段位表：**
```
青铜(0) → 白银(1000) → 黄金(5000) → 钻石(15000) → 王者(50000)
```

## 四、关键成功指标（KPI）

| 指标 | 目标值 | 意义 |
|------|--------|------|
| 首次留存 | 60% 用户玩第二次 | 核心循环验证 |
| 平均游戏时长 | >3 分钟/局 | 参与度 |
| 分享率 | >10% 死亡后分享 | 病毒系数 |
| 视觉满意度 | NPS > 50 | 产品市场匹配 |

## 五、与 Slither.io 的核心差异

| 维度 | Slither.io | 我们的方案 |
|------|------------|------------|
| 视觉 | 2D 扁平 | 3D + PBR + 粒子 + 后处理 |
| 沉浸感 | 一般 | 电影级视觉反馈 |
| 操作 | 尚可 | 鼠标跟随（更直觉） |
| 目标感 | 弱 | 段位 + 成就 + 收集 |
| 炫耀机制 | 分数分享 | 皮肤 + 视觉特效截图 |

## 六、风险与缓解

| 风险 | 概率 | 缓解方案 |
|------|------|----------|
| 3D 性能问题 | 中 | Canvas 大小自适应，降低特效保帧率 |
| 用户觉得太复杂 | 低 | 3 秒自动引导，只教移动 |
| 没有网络效应 | 低 | Phase 1 单机足够好玩 |
